# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest `main` | ✅ |
| older tags    | ❌ |

## Reporting a Vulnerability

Please **do not open a public issue** for security vulnerabilities.

Instead, use GitHub's private vulnerability reporting: go to the
repository's **Security** tab → **Report a vulnerability**. This sends
the report privately to the maintainer.

You can expect:

- An acknowledgement within 7 days
- An assessment and fix timeline within 30 days for confirmed issues
- Credit in the fix's release notes (unless you prefer otherwise)

## Scope Notes

Exiqus is no longer operated as a hosted service; this policy covers
the open-source codebase itself. Reports about self-hosted deployments
misconfigured by their operators (e.g. default passwords not changed)
are out of scope, but hardening suggestions are welcome as regular
issues.
